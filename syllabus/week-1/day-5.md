### 2. Build Content Generator (60 min)

**Create the database table:**
```
You: Create a content table in Supabase for my content engine.

Store:
- id, user_id, created_at (standard)
- topic: what the content is about
- blog_post: the full generated blog post (markdown)
- social_snippets: array of 3-5 tweetable quotes
- email_draft: newsletter version
- status: draft, reviewed, published

Include RLS so users only see their own content.
```

**Build the blog generator:**
```
You: Create a generateBlogPost function that:

1. Takes a topic as input
2. Reads CLAUDE.md for brand voice and audience info
3. Calls Gemini with a prompt that generates:
   - Attention-grabbing headline
   - Hook that identifies the problem
   - 3 main sections with actionable advice
   - Call-to-action relevant to [my product]
4. Returns markdown-formatted blog post
5. Targets 800-1200 words

The prompt should make the AI write like our brand‚Äî[describe your tone].
Use my blog-writer skill as reference for the structure.
```

**Test it:**
```
You: Add a test endpoint at /api/test-content that generates
a blog post about "[topic in your niche]" and returns it.
```

Check the output. Is it in your voice? Does it sound like your brand?

**Iterate on the prompt:**
```
You: The output is too [generic/formal/casual].
Adjust the prompt to [specific change].
Also make sure it mentions [your product] naturally, not forced.
```

### 3. Add Social + Email (75 min)

**Build social snippet generator:**
```
You: Create a generateSocialSnippets function that:

1. Takes a blog post as input
2. Extracts 5 shareable quotes/insights
3. Each snippet should be:
   - Under 280 characters (tweet-length)
   - Standalone (makes sense without context)
   - Intriguing (makes people want to read more)
4. Returns an array of 5 snippets

Format each with relevant emoji and hashtag suggestions.
```

**Build email draft generator:**
```
You: Create a generateEmailDraft function that:

1. Takes a blog post as input
2. Creates a newsletter version that:
   - Has a compelling subject line
   - Opens with a personal hook
   - Summarizes the key insights (not the full post)
   - Includes a "Read the full post" CTA
   - Ends with a P.S. that teases next week
3. Targets 200-300 words (email-appropriate length)

Make it feel personal, like I'm writing to a friend who's interested in [your niche].
```

**Create the full pipeline:**
```
You: Create a generateAllContent function that:

1. Takes a topic
2. Generates the blog post
3. Generates social snippets from the blog post
4. Generates email draft from the blog post
5. Saves everything to the content table
6. Returns the complete content object

Add proper error handling‚Äîif one step fails, save what we have
and mark which parts failed.
```

## Afternoon: Dashboard + Demo

### 4. Content Dashboard (60 min)

**Build the UI:**
```
You: Create a /content page for managing my content engine.

Include:
1. Topic input form
   - Text field for topic
   - "Generate Content" button
   - Loading state that shows progress ("Generating blog post...", "Creating snippets...", etc.)

2. Content list
   - Show all generated content
   - Display topic, status, created date
   - Click to expand and see full content

3. Content detail view
   - Show blog post (rendered markdown)
   - Show social snippets (each in a card, with copy button)
   - Show email draft
   - Edit capability for each section
   - "Mark as Reviewed" and "Publish" buttons

Match our brand styling. Make it feel like a real content management tool.
```

**Add the generate flow:**
```
You: When user submits a topic:

1. Show a progress modal with steps:
   - "Generating blog post..." (spinner)
   - "Creating social snippets..."
   - "Drafting email..."
   - "Saving to database..."
2. Each step shows checkmark when complete
3. On success, close modal and show the new content
4. On error, show which step failed and allow retry

Make it feel satisfying to watch the AI work.
```

### 5. Polish + Edge Cases (30 min)

**Handle edge cases:**
```
You: Add these improvements to the content engine:

1. Prevent duplicate topics (warn if similar topic exists)
2. Add "Regenerate" button for each content piece
3. Add character count to social snippets
4. Add "Copy to clipboard" for email draft
5. Show word count on blog post
6. Add delete content functionality

Also add empty state when no content exists yet.
```

**Final test:**
- Generate content for 2-3 different topics
- Verify all pieces are created
- Test edit and save
- Test on mobile
- Check that RLS works (other users can't see your content)

### Demo Script Template

```

"Creating content for [my product] used to take hours.

GENERATE (60 seconds):

"Now it's extracting shareable quotes..."
"And drafting a newsletter..."
[Content appears]

OUTPUT (30 seconds):
"Here's an 1000-word blog post, ready to publish."
[Scroll through it]
"5 tweets I can post right now."
[Show snippets]
"And an email draft for my newsletter."

CLOSE (15 seconds):
"5 days ago I had an idea. Now I have a product with users,
AI features, and a content machine. That's code:zero."
```

### Demo Day Rules

1. **2 minutes MAX.** Timer cuts you off.
2. **Live generation.** Show the AI working in real-time.
3. **Real output.** Show actual generated content.
4. **Your URL.** Demo from your deployed site.
5. **Have backup.** Pre-generated content if live fails.

### Presenting Order

Random order. When your name is called:
1. Share screen
2. State your name and product
3. Start timer
4. Demo
5. Stop when time's up

### Giving Feedback

For each demo, note:
1. **One thing that impressed you**
2. **One suggestion for improvement**

Be specific. "The AI output looked professional" ‚Üí "The way it matched your brand voice in the email was impressive."
